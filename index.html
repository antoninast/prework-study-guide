HTML

    HTML (HyperText Markup Language) is the standard language used to create and structure content on the web. It defines the structure of web pages using tags to organize text, images, links, forms, and other elements. HTML provides the foundation for web content, while CSS and JavaScript enhance the design and functionality.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The div element is used as a container to group related content for layout or styling purposes.
  • The img element is used to display images on a webpage and requires a src attribute for the image source.
  • The a element creates hyperlinks and requires an href attribute for the link destination.
  • The ul element creates an unordered list, while the ol element creates an ordered list.
  • The form element is used to collect user input and can include fields like input, textarea, and select.
  • The button element creates a clickable button, often used in forms or for interactivity.
  • The header, footer, article, and section semantic tags improve accessibility and readability.
  • The span element is used to style a specific part of text inline without breaking the flow of the content.
  • The meta element within the head provides metadata like character encoding (charset) and viewport settings.
  • The table element is used to display tabular data and is structured with thead, tbody, and tfoot sections.

CSS

    CSS (Cascading Style Sheets) is not a programming language and is used to style and layout web pages, controlling elements like colors, fonts, and positioning. It works with HTML to define the look of a site and enables responsive design for different screen sizes. CSS helps separate content from design, making websites easier to maintain and update. To make the CSS code work, we need to import the CSS file in our HTML document.
  • The margin property indicates how much space we want around the outside of an element.
  • The paddingproperty indicates how much space we want around the content inside an element.
  • The color property sets the text color of an element.
  • The background-color property changes the background color of an element.
  • The margin property controls the space outside an element's border.
  • The padding property controls the space inside an element's border.
  • The font-family property specifies the font type used for text.
  • The border property defines the style, width, and color of an element's border.
  • The width and height properties set the dimensions of an element.
  • The display property determines how an element is displayed (e.g., block, inline, flex, grid).
  • The position property specifies how an element is positioned on the page (e.g., static, relative, absolute, fixed, sticky).
  • The z-index property determines the stack order of elements (higher values appear on top).
  • The text-align property aligns text horizontally within an element (e.g., left, center, right).
  • The box-shadow property adds shadow effects to an element's box.
  • The transition property adds smooth animations when property values change.
  • The flexbox layout is used for aligning and distributing space in a container with properties like justify-content and align-items.
  • The grid layout is used for creating complex, responsive designs with rows and columns.

Git

    Git is a distributed version control system that tracks changes in files, primarily used for managing code in software development. It allows multiple developers to collaborate efficiently by enabling them to work on different branches, make changes, and merge them seamlessly. Git keeps a history of all changes, making it easy to revert to previous versions or view the progress of a project over time. Because it's distributed, every developer has a complete copy of the repository, allowing them to work offline and sync changes later. Git is commonly used with platforms like GitHub, GitLab, or Bitbucket for hosting and managing code repositories.
  • git status: checks what branch we are currently on, including staged, unstaged, and untracked files.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git checkout <branch-name>: switches to the specified branch.
  • git init: initializes a new Git repository in the current directory.
  • git clone <repository-url>: creates a local copy of a remote repository.
  • git add <file>: adds the specified file to the staging area. Use git add . to stage all changes.
  • git commit -m "message": records changes to the repository with a descriptive commit message.
  • git push: uploads local repository changes to a remote repository.
  • git pull: fetches and integrates changes from a remote repository into the local repository.
  • git branch: lists all branches in the repository and highlights the current branch.
  • git merge <branch-name>: merges the specified branch into the current branch.
  • git log: displays the commit history of the repository.
  • git diff: Shows the differences between changes in the working directory and the repository.
  • git reset <file>: Removes the specified file from the staging area without deleting the changes.
  • git stash: Temporarily saves uncommitted changes to clean the working directory.
  • git remote -v: Displays the URLs of remote repositories associated with the project.
  • git tag: Creates a tag for a specific commit, often used for versioning.

JavaScript

    JavaScript is a high-level, dynamic programming language primarily used to create interactive effects within web browsers. It is one of the core technologies of the World Wide Web, alongside HTML and CSS, and is an essential tool for web development. It runs in the browser (client-side) but can also be used on the server (with Node.js). JavaScript enables features like form validation, animations, and event handling, making websites interactive. It is versatile, supporting both front-end and back-end development, and is essential for modern web applications.
  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Functions are one of the fundamental building blocks in JavaScript. A function definition consists of the function keyword, followed by:
    • The name of the function.
    • A list of parameters to the function, enclosed in parentheses and separated by commas.
    • The JavaScript statements that define the function, enclosed in curly braces, { /* … */ }.
  • The if...else statement executes a statement if a specified condition is truthy. If the condition is falsy, another statement in the optional else clause will be executed.
  • Loops offer a quick and easy way to do something repeatedly. A for loop repeats until a specified condition evaluates to false.
  • Arrays:
    • JavaScript arrays are resizable and can contain a mix of different data types.
    • JavaScript arrays are zero-indexed: the first element of an array is at index 0, the second is at index 1, and so on — and the last element is at the value of the array's length property minus 1.
